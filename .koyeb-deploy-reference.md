# Atlas API - Koyeb Deployment

Quick reference for deploying Atlas API to Koyeb.

## Environment Variables for Koyeb

Copy these to Koyeb's environment variables section:

```bash
# Application
PORT=8000
NODE_ENV=production

# Database (use your external PostgreSQL)
DATABASE_URL=postgresql://user:password@host:5432/database

# Supabase
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_KEY=your-supabase-key
```

## Important Notes

1. **Port**: Koyeb uses port `8000` by default (not `4000`)
2. **Database**: Must be external (Supabase, Neon, Railway, etc.)
3. **Secrets**: Mark `DATABASE_URL` and `SUPABASE_KEY` as secrets
4. **Health Check**: Set initial delay to 60-90 seconds for migrations

## Quick Deploy Steps

1. Push code to GitHub
2. Connect GitHub to Koyeb
3. Create Web Service → Select GitHub → Choose repo
4. Builder: Dockerfile
5. Add environment variables above
6. Port: 8000
7. Deploy!

## After Deployment

Your API will be available at:
```
https://atlas-api-YOUR-ORG.koyeb.app
```

API Documentation:
```
https://atlas-api-YOUR-ORG.koyeb.app/doc
```

See `KOYEB_DEPLOYMENT.md` for detailed instructions.
